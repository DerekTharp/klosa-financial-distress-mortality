# Code Review Top Findings

Timestamp: 2026-03-25 10:12 EDT

This is an interim memo with the highest-priority sensitivity/diagnostic mismatches and overclaims identified during the code review.

## Top 5

1. The Basic Pension sensitivity analysis excludes wave 9, even though the manuscript discusses the full 2006-2022 study period. The code defines post-2014 using `wave >= 5` but then runs the period-stratified and age-65+ analyses only on waves 5-8, omitting 2022. See [analysis/12_basic_pension_analysis.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/12_basic_pension_analysis.py#L42), [analysis/12_basic_pension_analysis.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/12_basic_pension_analysis.py#L68), [analysis/12_basic_pension_analysis.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/12_basic_pension_analysis.py#L188), and the manuscript claim at [submission_ssm/manuscript.md](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/submission_ssm/manuscript.md#L143).

2. The wealth-shock post hoc power calculation is too cleanly stated for what the code actually does. It uses a simple detectable-HR formula based on person-interval counts from the time-varying dataset, which is a rough approximation rather than a publication-grade power calculation for a repeated-interval Cox setting. See [analysis/07_extended_sensitivity.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/07_extended_sensitivity.py#L553), [analysis/07_extended_sensitivity.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/07_extended_sensitivity.py#L574), and the manuscript wording at [submission_ssm/manuscript.md](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/submission_ssm/manuscript.md#L103).

3. The manuscript says unpenalized time-varying results are reported for comparison, but the regularization check is narrower than that sentence implies. The explicit penalizer sensitivity in the code covers low economic satisfaction and wealth shock only, not low income or the combined time-varying model that the manuscript highlights. See [analysis/07_extended_sensitivity.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/07_extended_sensitivity.py#L366), [analysis/08_objective_vs_subjective.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/08_objective_vs_subjective.py#L387), and [submission_ssm/manuscript.md](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/submission_ssm/manuscript.md#L71).

4. The time-partitioned “early vs late follow-up” sensitivity is more exploratory than the manuscript presentation suggests. The code creates separate 0-8 and 8-16 year models by filtering survivors and resetting follow-up time, which is useful descriptively but not a clean formal test of time-varying coefficients. See [analysis/07_extended_sensitivity.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/07_extended_sensitivity.py#L321) and the manuscript interpretation at [submission_ssm/manuscript.md](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/submission_ssm/manuscript.md#L107).

5. The measurement-sensitivity analyses run on a smaller, augmented-covariate sample than the main baseline health-adjusted model, but that sample change is not surfaced in the manuscript. The cut-point and spline models pull in wave-2 drinking and exercise covariates and run on `N=8441`, not the larger primary-model sample. See [analysis/10_measurement_sensitivity.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/10_measurement_sensitivity.py#L35), [analysis/10_measurement_sensitivity.py](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/analysis/10_measurement_sensitivity.py#L86), [output/supplementary/measurement_sensitivity.json](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/output/supplementary/measurement_sensitivity.json), and [submission_ssm/manuscript.md](/Users/derektharp/Documents/_Projects/26.21 Korean Financial Distress -- Mortality/submission_ssm/manuscript.md#L99).
